<h2>
    <strong>generate_tfmovr v1.0 by Ikotas</strong>
</h2>
<p>
    <strong>Description</strong>
</p>
<p>
    This tool processes and cleans up TFM output files. It analyzes telecine (interlace) cycles and corrects them into consistent, clean patterns. By eliminating incorrect match code (c/p) detections, it enables high-precision deinterlacing.
</p>
<p>
    <strong>Target Sources</strong>
</p>
<ul>
    <li data-list-item-id="ee98eebaf8fd2a63999c58601ca874fa3">
        Sources telecined from 23.976 fps to 29.97 fps.
    </li>
    <li>
        Live-action and CG animation.
        <i>*Note: Hand-drawn (cel) animation is not supported as its irregular cycles are difficult to analyze.</i>
    </li>
</ul>
<p>
    <strong>Required Files</strong>
</p>
<ul>
    <li>
        An output file generated by TFM.
    </li>
    <li>
        Recommended TFM settings for output:<br>
        <code>TFM(mode=0,pp=1,slow=2,micmatching=0,output="x:\path\filename.tfm")</code>
    </li>
</ul>
<p>
    <strong>Usage</strong>
</p>
<p>
    <code>generate_tfmovr.exe [options] TFM_output_file(*.tfm)</code>
</p>
<p>
    <strong>Options</strong>
</p>
<ul>
    <li>
        <code>-d &lt;num&gt;</code> : <strong>dominantTh</strong> (Default: 6)
    </li>
    <li>
        <code>-m &lt;num&gt;</code> : <strong>maintainTh</strong> (Default: 3)
    </li>
    <li>
        <code>-a &lt;num&gt;</code> : <strong>adoptionTh</strong> (Default: 4)
    </li>
    <li>
        <code>-c &lt;num&gt;</code> : <strong>consecutTh</strong> (Default: 2)
    </li>
    <li>
        <code>-o &lt;file&gt;</code> : <strong>output filename</strong>
    </li>
</ul>
<p>
    --------------------------------------------------------------------------------
</p>
<p>
    <strong>Option Details</strong>
</p>
<p>
    <strong>-d &nbsp;: dominantTh (Dominance Override Threshold)</strong>
</p>
<ul>
    <li>
        <strong>Purpose:</strong> Controls "forced transitions" for clear scene changes or overwhelmingly stable new cycles.
    </li>
    <li>
        <strong>Default:</strong> 6
    </li>
    <li>
        <strong>Details:</strong> Triggered when a new pattern candidate achieves a <strong>Strict Match</strong> (all 5 frames match the pattern) in 6 or more groups out of 10 (within a 50-frame look-ahead). When this threshold is met, the tool forces a transition even if the current cycle still meets maintenance conditions.
    </li>
    <li class="ck-list-marker-bold">
        <strong>Adjustment:</strong>
        <ul>
            <li>
                Lowering the value makes the tool more sensitive to scene changes but may increase reactions to temporary noise.
            </li>
            <li>
                Raising the value ensures the tool only changes cycles when there is very strong evidence.
            </li>
        </ul>
    </li>
</ul>
<p>
    <strong>-m &nbsp;: maintainTh (Maintenance Threshold)</strong>
</p>
<ul>
    <li>
        <strong>Purpose:</strong> Determines the "persistence" of the currently active cycle.
    </li>
    <li>
        <strong>Default:</strong> 3
    </li>
    <li>
        <strong>Details:</strong> Evaluates the <strong>MatchCore score</strong> (checks if 'c' is incorrectly mixed into 'p' positions) for the current Identity. If at least 3 out of 10 groups match, the cycle is considered "alive" and maintained.
    </li>
    <li class="ck-list-marker-bold">
        <strong>Adjustment:</strong>
        <ul>
            <li>
                Lowering the value strengthens cycle persistence, keeping the same cycle even with jittery data.
            </li>
            <li>
                Raising the value causes the tool to abandon a cycle faster if the data begins to deviate.
            </li>
        </ul>
    </li>
</ul>
<p>
    <strong>-a &nbsp;: adoptionTh (Weak Adoption Threshold)</strong>
</p>
<ul>
    <li>
        <strong>Purpose:</strong> Sets the criteria for adopting a new pattern after the current one has collapsed.
    </li>
    <li>
        <strong>Default:</strong> 4
    </li>
    <li>
        <strong>Details:</strong> Used when the current cycle score falls below <code>-m</code> (collapse state). If a new candidate has a <strong>Strict Match score of 4 or higher</strong> out of 10, it is adopted as the new starting point.
    </li>
    <li class="ck-list-marker-bold">
        <strong>Adjustment:</strong>
        <ul>
            <li>
                Lowering the value makes the tool more aggressive in assigning cycles in unstable sections (higher noise risk).
            </li>
            <li>
                Raising the value prefers <code>n/c</code> (progressive) processing unless a strong telecine match is found.
            </li>
        </ul>
    </li>
</ul>
<p>
    <strong>-c &nbsp;: consecutTh (Consecutive Match Threshold)</strong>
</p>
<ul>
    <li>
        <strong>Purpose:</strong> Salvages "extremely short cycles" or sudden phase shifts that cumulative scores might miss.
    </li>
    <li>
        <strong>Default:</strong> 2
    </li>
    <li>
        <strong>Details:</strong> If a new candidate matches perfectly (<strong>Strict Match</strong>) for this many times <strong>consecutively</strong>, it is adopted immediately regardless of the cumulative score. This allows the tool to accurately pick up short telecine segments lasting only 10–20 frames.
    </li>
    <li>
        <strong>Adjustment:</strong>
        <ul>
            <li>
                Setting this to 1 makes the cycle detection extremely unstable.
            </li>
            <li>
                Raising the value ignores short interruptions and focuses on long, stable cycles.
            </li>
        </ul>
    </li>
</ul>
<p>
    <strong>-o &nbsp;: Output Filename Specification</strong>
</p>
<ul>
    <li>
        <strong>Purpose:</strong> Customizes the name of the generated result file.
    </li>
    <li class="ck-list-marker-bold">
        <strong>Details:</strong>
        <ul>
            <li>
                With extension (e.g., <code>-o my_list.txt</code>): Outputs exactly as named.
            </li>
            <li>
                Without extension (e.g., <code>-o my_list</code>): Automatically appends <code>.tfmovr</code>.
            </li>
            <li>
                Not specified: Replaces the input <code>.tfm</code> extension with <code>.tfmovr</code>.
            </li>
        </ul>
    </li>
</ul>
<p>
    --------------------------------------------------------------------------------
</p>
<p>
    <strong>Setting Guidelines</strong>
</p>
<ul>
    <li>
        <strong>To capture scene changes more reliably:</strong> Lower <code>-d</code> to 5.
    </li>
    <li>
        <strong>To stabilize cycles in noisy sources:</strong> Lower <code>-m</code> to 2 and raise <code>-a</code> to 5.
    </li>
    <li>
        <strong>To prioritize short cycle detection:</strong> Keep <code>-c</code> at 2 (default) or tighten cumulative thresholds to rely primarily on consecutive matches.
    </li>
</ul>
<p>
    --------------------------------------------------------------------------------
</p>
<p>
    <strong>Encoding Settings</strong>
</p>
<p>
    <code>deinted=nnedi3(field=-1)</code><br>
    <code>TFM(mode=0,pp=3,MI=200,clip2=deinted,ovr="xxxxx.tfmovr")</code>
</p>
<p>
    <i>*Note: Only for the pcccp pattern, the first frame may be striped.</i><br>
    <i>Since this is a scene change frame, MI=256 might be acceptable.</i>
</p>
<p>
    <strong>For sources with hardsubs:</strong>
</p>
<p>
    <code>deinted=nnedi3(field=-1)</code><br>
    <code>TFM(mode=0,pp=3,MI=150,clip2=deinted,ovr="xxxxx.tfmovr")</code>
</p>
<p>
    <i>*Note: Setting MI around 150 is recommended to ensure the deinterlacer reacts primarily to subtitles and not to background noise.</i>
</p>

<h2>
    <strong>convert_tfmovr2se.bat v1.0 by Ikotas</strong>
</h2>
<p>
    <strong>Description</strong>
</p>
<p>
    This batch script reads the <code>.tfmovr</code> (or processed <code>.tfm</code>) file and converts each entry into <code>SelectEvery</code> commands, outputting a <code>.se</code> file.
</p>
<p>
    Since telecine phase changes do not always occur at multiples of 5, this approach handles cases that <code>TDecimate</code> cannot process correctly (e.g., preventing orphaned or duplicate frames during phase shifts).
</p>
<p>
    <strong>Execution</strong>
</p>
<p>
    <code>convert_tfmovr2se.bat generate_tfmovr_output_file(*.tfmovr)</code>
</p>
<p>
    <strong>Encoding Settings</strong>
</p>
<p>
    <code>deinted=nnedi3(field=-1)</code><br>
    <code>TFM(mode=0,pp=3,MI=200,clip2=deinted,ovr="xxxxx.tfmovr")</code><br>
    <code>Import("xxxxx.se")</code>
</p>
<p>
    <strong>Caution</strong>
</p>
<p>
    Using <code>SelectEvery</code> may result in a slight increase or decrease in the total frame count.
</p>

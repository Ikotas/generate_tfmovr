<h2><strong>generate_tfmovr.exe</strong></h2>

<p><strong>Description</strong></p>

<p>This tool processes and cleans up TFM output files. It analyzes telecine (interlace) cycles and corrects them into consistent, clean patterns.<br />
By eliminating incorrect match code (c/p) detections, it enables accurate field matching.</p>

<p><strong>Target Sources</strong></p>

<ul>
	<li>Sources telecined from 23.976 fps to 29.97 fps.</li>
	<li>Live-action and CG animation.<br />
	<i>*Note: Hand-drawn (cel) animation is not supported as its irregular cycles are difficult to analyze.</i></li>
</ul>

<p><strong>Required Files</strong></p>

<ul>
	<li>An output file generated by TFM.</li>
	<li>Recommended TFM settings for output:<br />
	<code>TFM(mode=0,pp=1,slow=2,micmatching=0,output=&quot;x:\path\filename.tfm&quot;)</code><br />
	<i>*Note: pp=1 is required.</i></li>
</ul>

<p><strong>Usage</strong></p>

<p><code>generate_tfmovr.exe [options] TFM_output_file</code></p>

<p><strong>Options</strong></p>

<ul>
	<li><code>-d &lt;int&gt;</code> : <strong>dominantTh</strong> (Default: 6)</li>
	<li><code>-m &lt;int&gt;</code> : <strong>maintainTh</strong> (Default: 3)</li>
	<li><code>-a &lt;int&gt;</code> : <strong>adoptionTh</strong> (Default: 5)</li>
	<li><code>-c &lt;int&gt;</code> : <strong>consecutTh</strong> (Default: 3)</li>
	<li><code>-p &lt;int&gt;</code> : p<strong>rogressTh</strong> (Default: 95)</li>
	<li><code>-r &lt;int&gt;</code> : <strong>progressRa</strong> (Default: 150)</li>
	<li><code>-o &lt;file&gt;</code> : <strong>output filename</strong></li>
</ul>

<hr>

<p><strong>Option Details</strong></p>

<p><strong>-d &nbsp;: dominantTh (Dominance Override Threshold)</strong></p>

<ul>
	<li><strong>Purpose:</strong> Controls &quot;forced transitions&quot; for clear scene changes or overwhelmingly stable new cycles.</li>
	<li><strong>Default:</strong> 6</li>
	<li><strong>Details:</strong> Triggered when a new pattern candidate achieves a <strong>Strict Match</strong> (all 5 frames match the pattern) in 6 or more groups out of 10 (within a 50-frame look-ahead).<br />
	When this threshold is met, the tool forces a transition even if the current cycle still meets maintenance conditions.</li>
	<li class="ck-list-marker-bold"><strong>Adjustment:</strong>
	<ul>
		<li>Lowering the value makes the tool more sensitive to scene changes but may increase reactions to temporary noise.</li>
		<li>Raising the value ensures the tool only changes cycles when there is very strong evidence.</li>
	</ul>
	</li>
</ul>

<p><strong>-m &nbsp;: maintainTh (Maintenance Threshold)</strong></p>

<ul>
	<li><strong>Purpose:</strong> Determines the &quot;persistence&quot; of the currently active cycle.</li>
	<li><strong>Default:</strong> 3</li>
	<li><strong>Details:</strong> Evaluates the <strong>MatchCore score</strong> (checks if <code>c</code> is incorrectly mixed into <code>p</code> positions) for the current Identity. If at least 3 out of 10 groups match, the cycle is considered &quot;alive&quot; and maintained.</li>
	<li class="ck-list-marker-bold"><strong>Adjustment:</strong>
	<ul>
		<li>Lowering the value strengthens cycle persistence, keeping the same cycle even with jittery data.</li>
		<li>Raising the value causes the tool to abandon a cycle faster if the data begins to deviate.</li>
	</ul>
	</li>
</ul>

<p><strong>-a &nbsp;: adoptionTh (Weak Adoption Threshold)</strong></p>

<ul>
	<li><strong>Purpose:</strong> Sets the criteria for adopting a new pattern after the current one has collapsed.</li>
	<li><strong>Default:</strong> 4</li>
	<li><strong>Details:</strong> Used when the current cycle score falls below <code>-m</code> (collapse state).<br />
	If a new candidate has a <strong>Strict Match score of 4 or higher</strong> out of 10, it is adopted as the new starting point.</li>
	<li class="ck-list-marker-bold"><strong>Adjustment:</strong>
	<ul>
		<li>Lowering the value makes the tool more aggressive in assigning cycles in unstable sections (higher noise risk).</li>
		<li>Raising the value prefers <code>c</code> (progressive) processing unless a strong telecine match is found.</li>
	</ul>
	</li>
</ul>

<p><strong>-c &nbsp;: consecutTh (Consecutive Match Threshold)</strong></p>

<ul>
	<li><strong>Purpose:</strong> Salvages &quot;extremely short cycles&quot; or sudden phase shifts that cumulative scores might miss.</li>
	<li><strong>Default:</strong> 2</li>
	<li><strong>Details:</strong> If a new candidate matches perfectly (<strong>Strict Match</strong>) for this many times <strong>consecutively</strong>, it is adopted immediately regardless of the cumulative score. This allows the tool to accurately pick up short telecine segments lasting only 10&ndash;20 frames.</li>
	<li><strong>Adjustment:</strong>
	<ul>
		<li>Setting this to 1 makes the cycle detection extremely unstable.</li>
		<li>Raising the value ignores short interruptions and focuses on long, stable cycles.</li>
	</ul>
	</li>
</ul>

<p><strong>-p : progressTh (Progressive Detection Threshold)</strong></p>

<ul>
	<li><strong>Purpose:</strong> Sets the statistical percentage of <code>c</code> frames required to identify a section as progressive.</li>
	<li><strong>Default:</strong> 95 (%)</li>
	<li><strong>Details:</strong> In a range of <code>-r</code> frames, if the percentage of <code>c</code> frames is equal to or greater than this value, the tool considers the section for <code>c</code> transition.</li>
</ul>
<strong>-r : progressRa (Progressive Scan Range)</strong>

<ul>
	<li><strong>Purpose:</strong> Defines the look-ahead range for progressive detection and the jump interval.</li>
	<li><strong>Default:</strong> 150 (frames)</li>
	<li><strong>Details:</strong> This value is used both as a window for calculating the <code>c</code> frame percentage and as the minimum duration for a progressive <code>c</code> entry.</li>
</ul>
<strong>-o &nbsp;: Output Filename Specification</strong>

<ul>
	<li><strong>Purpose:</strong> Customizes the name of the generated result file.</li>
	<li class="ck-list-marker-bold"><strong>Details:</strong>
	<ul>
		<li>With extension (e.g., <code>-o my_list.txt</code>): Outputs exactly as named.</li>
		<li>Without extension (e.g., <code>-o my_list</code>): Automatically appends <code>.tfmovr</code>.</li>
		<li>Not specified: Replaces the input <code>.tfm</code> extension with <code>.tfmovr</code>.</li>
	</ul>
	</li>
</ul>

<hr>

<p><strong>Setting Guidelines</strong></p>

<ul>
	<li><strong>To capture scene changes more reliably:</strong> Lower <code>-d</code> to 5.</li>
	<li><strong>To stabilize cycles in noisy sources:</strong> Lower <code>-m</code> to 2 and raise <code>-a</code> to 6.</li>
	<li><strong>To prioritize short cycle detection:</strong> Lower&nbsp;<code>-c</code>&nbsp;to&nbsp;2 or tighten cumulative thresholds to rely primarily on consecutive matches.</li>
</ul>

<hr>

<p><strong>Encoding Settings</strong></p>

<p><code>deinted=nnedi3(field=-1)</code><br />
<code>TFM(mode=0,pp=3,MI=200,clip2=deinted,ovr=&quot;xxxxx.tfmovr&quot;)</code></p>

<p><i>*Note: Feel free to use your preferred deinterlacing plugin.<br />
With the pcccp pattern only, the first frame may appear striped.<br />
This also serves as a safeguard in case of omissions caused by irregular match codes.</i></p>

<p><strong>For sources with hardsubs:</strong></p>

<p><code>deinted=nnedi3(field=-1)</code><br />
<code>TFM(mode=0,pp=3,MI=150,clip2=deinted,ovr=&quot;xxxxx.tfmovr&quot;)</code></p>

<p><i>*Note: If the MI value is too low, the system will react to too many things, defeating the purpose of the configuration, so I recommend setting it to around 150 to ensure it responds primarily to subtitles.<br />
Please adjust the actual value as needed.</i></p>

<h2><strong>convert_tfmovr2se.bat</strong></h2>

<p><strong>Description</strong></p>

<p>This batch script reads the <code>.tfmovr</code> file and converts each entry into <code>SelectEvery</code> commands, outputting a <code>.se</code> file.</p>

<p>Since the timing of cycle changes is not limited to multiples of 5, using <code>SelectEvery</code> in <code>TDcimate</code> can prevent cases where data is missing or duplicated.</p>

<p><strong>Execution</strong></p>

<p><code>convert_tfmovr2se.bat generate_tfmovr_output_file(*.tfmovr)</code></p>

<p><strong>Encoding Settings</strong></p>

<p><code>deinted=nnedi3(field=-1)</code><br />
<code>TFM(mode=0,pp=3,MI=200,clip2=deinted,ovr=&quot;xxxxx.tfmovr&quot;)</code><br />
<code>Import(&quot;xxxxx.se&quot;)</code></p>

<p><strong>Caution</strong></p>

<p>Using <code>SelectEvery</code> may result in a slight increase or decrease in the total frame count.</p>

<h2>
    <strong>convert_tfmovr2bookmark.bat</strong>
</h2>

<p><strong>Description</strong></p>

<p>This batch script reads the <code>.tfmovr</code> file and outputs the starting frame number for each line to a <code>.log</code> file.<br>
    Open the .log file via the AvsPmod menu (Macros &gt; Import bookmarks from file), then press F2 or Shift+F2 to sequentially view the starting frames of the cycles output to the <code>.tfmovr</code> file.</p>

<p><strong>Execution</strong></p>

<p><code>convert_tfmovr2bookmark.bat generate_tfmovr_output_file(*.tfmovr)</code></p>
